## 1. Module Title

**Git & GitHub**

---

## 2. Module Description

This module introduces Git and GitHub as **core development skills** required for modern software development.  
It focuses on version control fundamentals, collaboration workflows, and safe code management practices used in real-world teams.

The module is designed to help learners understand **why version control is needed**, not just how to run commands.

---

## 3. Module Structure Overview

### Total Topics

**5 Topics**

### Total Learning Objectives (LOs)

**15 Learning Objectives**

> Note: Topics 04 and 05 are **skippable**, depending on course requirements and available time.

---

## 4. Target Audience

This module is intended for:

- Learners who already know basic coding
- Students or professionals with **at least 2–3 months of coding experience**
- Anyone who has written code and wants to understand version control and collaboration
- Learners preparing for real-world or team-based software development

---

## 5. Educational Background

Learners are expected to have:

- Basic programming knowledge in any language
- Familiarity with files, folders, and code editors
- Basic understanding of how programs are written and executed

Formal computer science education is **not mandatory**.

---

## 6. Experience Level

This module is suitable for:

- **Beginner to early-intermediate** learners
- Developers with **2–6 months of coding experience**
- Learners new to collaborative development workflows

---

## 7. Domain / Field

This module falls under:

- Software Development
- Version Control Systems
- Collaborative Development Practices
- DevOps Foundations (introductory level)

---

## 8. Prerequisites

Before starting this module, learners should be comfortable with:

- Writing basic code
- Using a code editor (VS Code or similar)
- Running simple terminal or command-line commands

This module is **not recommended for absolute beginners** with no coding experience.

---

## 9. Learning Expectations

Learners are expected to:

- Practice Git commands regularly
- Work with real repositories
- Commit code frequently with meaningful messages
- Use branches instead of working directly on main
- Learn from mistakes such as merge conflicts and rollback scenarios

Hands-on practice is essential for successful completion.

---

## 10. Learning Objectives (Topic-wise)

### Topic 01: Git Installation & Core Version Control

**(3 LOs – Non-Skippable)**

1. Describe version control and its importance in software development
2. Install Git and configure user identity
3. Track, manage, and safely undo file changes

---

### Topic 02: GitHub Basics

**(3 LOs – Non-Skippable)**

4. Configure and synchronize local and remote repositories
5. Push and pull code using GitHub
6. Collaborate using pull requests and repository access control

---

### Topic 03: Branching & Collaboration

**(3 LOs – Non-Skippable)**

7. Develop features using branches
8. Resolve merge conflicts effectively
9. Improve commit history using basic rebase

---

### Topic 04: Project Management using GitHub

**(3 LOs – Skippable)**

> Include this topic **only if the course aims to teach GitHub as a core skill and sufficient time is available**.

10. Track project work using GitHub Issues and Milestones
11. Organize teamwork using Project Boards
12. Follow commit conventions for team collaboration

---

### Topic 05: CI-CD & GitHub Actions

**(3 LOs – Skippable)**

> This topic is **fully optional** and should be included only for extended or advanced courses.

13. Understand CI-CD automation within GitHub
14. Build basic build/test automation using GitHub Actions
15. Configure Secrets, Environment Variables, and enforce Branch Protection

---

## 12. Frameworks & Libraries

This module is **language-agnostic** and does not rely on any specific framework or library.

---

## 13. Special Instructions

Learners must:

- Avoid using Git commands without understanding their purpose
- Follow proper commit message practices
- Use branches for feature development
- Actively resolve conflicts instead of avoiding them

---

## 14. Assessment & Evaluation

Assessment may include:

- Practical Git exercises
- Repository-based assignments
- Collaboration tasks using GitHub
- Workflow demonstrations

Evaluation focuses on **correct usage, understanding, and discipline**.

---

## 15. Completion Criteria

A learner is considered to have completed this module when they:

- Complete all non-skippable topics (Topics 01–03)
- Demonstrate safe and correct Git usage
- Collaborate using GitHub repositories
- Follow standard Git workflows consistently

---

## 16. Additional Notes

- GitHub is treated as a **core professional skill**, not just a tool
- Topics 04 and 05 should be included **only if course objectives demand deeper GitHub expertise**
- For basic Git-focused courses, Topics 01–03 are sufficient
