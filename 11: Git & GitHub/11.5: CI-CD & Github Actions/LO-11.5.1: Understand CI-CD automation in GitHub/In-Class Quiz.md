**Q1. What is the primary purpose of using CI/CD in software projects?**

A) To replace the need for developers to write tests

B) To automatically validate code changes and prevent unstable code from being merged

C) To manually deploy applications faster

D) To manage issues and project boards

E) Topic not covered

**Correct Answer: B**

---

**Q2. Which of the following best describes Continuous Integration (CI)?**

A) Automatically deploying code to production without approval

B) Writing code only at the end of the project

C) Frequently integrating small code changes and validating them with automated checks

D) Running tests only before final release

E) Topic not covered

**Correct Answer: C**

---

**Q3. What happens when a CI/CD pipeline check fails on a Pull Request?**

A) The Pull Request is automatically merged

B) The issue linked to the PR is closed

C) The PR cannot be merged until the issues are fixed

D) The pipeline skips remaining stages and deploys anyway

E) Topic not covered

**Correct Answer: C**
