**Q1. What is the primary goal of using `git rebase` instead of `git merge`?**
A) To delete old commits
B) To preserve all branch structures
C) To create a cleaner, linear commit history
D) To automatically resolve conflicts
E) Topic not covered
**Correct Answer: C**

---

**Q2. In which situation is it safest to use rebase?**
A) On the shared `main` branch
B) On a feature branch before it is pushed
C) After a Pull Request is merged
D) On any branch with collaborators
E) Topic not covered
**Correct Answer: B**

---

**Q3. Why should rebased branches be pushed carefully to a remote repository?**
A) Rebase deletes files permanently
B) Rebase slows down Git operations
C) Rebase rewrites commit history and changes commit IDs
D) Rebase disables merge functionality
E) Topic not covered
**Correct Answer: C**
