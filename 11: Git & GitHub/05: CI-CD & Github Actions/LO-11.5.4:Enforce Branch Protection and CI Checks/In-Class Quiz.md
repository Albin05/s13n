**Q1. What is the primary purpose of enabling Branch Protection on a main branch?**

A) To speed up merging by skipping reviews

B) To allow developers to push directly to main

C) To enforce rules so that only verified and reviewed code is merged

D) To replace CI/CD pipelines

E) Topic not covered

**Correct Answer: C**

---

**Q2. What does “CI enforcement” mean in the context of branch protection?**

A) CI workflows run only after merging

B) CI results are optional and can be ignored

C) Code cannot be merged unless all required CI checks pass

D) CI runs only on local machines

E) Topic not covered

**Correct Answer: C**

---

**Q3. What happens when a required status check fails on a Pull Request with branch protection enabled?**

A) The Pull Request is automatically closed

B) The code is merged with a warning

C) The merge button is disabled until the issue is fixed

D) The branch protection rule is removed

E) Topic not covered

**Correct Answer: C**
