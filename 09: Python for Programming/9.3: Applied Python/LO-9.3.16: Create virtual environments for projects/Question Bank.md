## Question Bank: Create Virtual Environments

---

### Q1: Create and Activate (3 minutes, Low Difficulty)
Write the commands to: create a venv named "myenv", activate it (Mac and Windows), verify activation, deactivate.

**Key Concepts:** python -m venv, activate, deactivate

---

### Q2: Project Setup (3 minutes, Low Difficulty)
Create a complete project setup: create venv, activate, install packages (requests, flask), generate requirements.txt, add .gitignore for venv.

**Key Concepts:** Full venv workflow

---

### Q3: Multiple Environments (5 minutes, Medium Difficulty)
Explain when and why you'd need separate virtual environments. Give 3 scenarios where projects need different dependencies.

**Key Concepts:** Dependency isolation, project separation

---

### Q4: Reproduce an Environment (5 minutes, Medium Difficulty)
Write a script that reads requirements.txt and verifies all packages are installed at correct versions. Report any mismatches.

**Key Concepts:** Environment verification

---

### Q5: Environment Troubleshooting (5 minutes, Medium Difficulty)
List 5 common venv problems and their solutions: wrong Python version, packages not found after activation, permission errors, IDE not using venv, etc.

**Key Concepts:** Common venv issues
